# Data protection impact assessment draft

Phase 2 processes seeded patient-identifiable inpatient nursing records plus read-only snapshots from seeded sibling systems. It exercises access control, purpose-of-use, minimisation, provenance, reconciliation, shared-screen privacy, retention, correction, and breach-investigation controls. HMIS receives only approved ward counts. The seeded people are fictional and marked as such. A production deployment requires controller-specific lawful basis, country overlay, processor and transfer assessment, retention schedule, patient notice, and DPO approval.

For the approved synthetic clinical-simulation scope, the Agent Clinical Safety Officer reviewed only governance metadata and de-identified assurance evidence. The human release authority approved that exact fictional-data scope. This is not a controller-specific DPIA sign-off and cannot be inherited by a live-patient deployment.
